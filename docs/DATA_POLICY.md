# Data Policy

This repository is prepared for open-source sharing without exposing the private research corpus.

## Public Data

`data/sample/` contains only synthetic records generated by `python -m zpaper.make_sample_data`.

The sample CSV mirrors the analysis schema:

| Column | Meaning |
| --- | --- |
| `journal`, `SO` | Journal code and source title |
| `DI` | DOI-like identifier; synthetic in public data |
| `AU`, `AUNUM` | Author display string and author count |
| `PY`, `VL`, `IS` | Publication year, volume, issue |
| `cd_index`, `n_total`, `n_disruptive`, `n_consolidating` | Citation-disruption outcome fields |
| `Mean`, `Top5_Mean`, `IMEAN`, `MMEAN`, `RMEAN`, `DMEAN`, `ABMEAN`, `TIMEAN` | Semantic visibility scores |
| `Skewness`, `Frac_Below_Global_Q25`, `Frac_Above_Global_Q75` | Distributional semantic descriptors |
| `PP`, `FIS`, `JIF`, `NR`, `2024JIF`, `EF` | Paper and journal controls |

The sample metadata JSON mirrors the private metadata structure:

`Title`, `Introduction`, `Methods`, `Results`, `Discussion`, `TI`, `AB`, `SO`, `AU`, `DT`, `PY`, `VL`, `IS`, `DI`.

## Private Data

Do not commit these local assets:

- `paper_dataset3.csv`
- `USEDATA/`
- `outputs/`
- `preresults/`
- `redraw/`
- `Target journal/`
- root-level `.docx`, `.pdf`, `.png`, and working `.txt` files
- `scripts/` from the original local workspace, because some scripts contain manuscript prose and private working paths

For private reruns, put source files under `data/private/` or pass paths outside the Git repository.

## Release Checklist

Before pushing to GitHub:

1. Run `git status --short` and confirm only sanitized files are staged.
2. Confirm `git check-ignore -v paper_dataset3.csv USEDATA outputs preresults scripts` reports ignore rules.
3. Regenerate demo data with `python -m zpaper.make_sample_data`.
4. Run `python -m zpaper.semantic_disruption`.
5. Inspect staged files with `git diff --cached --stat`.
